# vps-security-bootstrap

[English README](./README.md)

`vps-security-bootstrap` — это интерактивный Bash-скрипт для первичной защиты нового VPS. Первая версия рассчитана на `Ubuntu 24.04 LTS`, но структура репозитория специально сделана нейтральной, чтобы позже можно было добавить поддержку других ОС.

Скрипт в начале предлагает выбрать язык (`English / Русский`), а затем пошагово проводит пользователя через обновление системы, создание sudo-пользователя, настройку SSH, UFW и опциональную установку Fail2Ban.

## Что делает скрипт

- Обновляет систему через `apt update` и `apt upgrade -y`.
- Определяет, рекомендуется ли перезагрузка, и спрашивает подтверждение перед reboot.
- При запуске с root-доступом может предложить сменить пароль `root`.
- Создает или использует существующего администратора без root и подтверждает права sudo.
- Настраивает SSH через отдельный drop-in конфиг в `/etc/ssh/sshd_config.d/`.
- Поддерживает режим только по ключу или вариант с сохранением входа по паролю.
- Настраивает UFW в безопасном порядке при смене SSH-порта.
- Устанавливает и настраивает Fail2Ban через локальный конфиг.
- В конце выводит итоговую сводку и команду для проверки нового SSH-подключения.

## Что меняет скрипт

- При необходимости устанавливает пакеты: `openssh-server`, `ufw`, `fail2ban`.
- Пишет SSH-настройки в `/etc/ssh/sshd_config.d/99-vps-security-bootstrap.conf`.
- Пишет настройки Fail2Ban в `/etc/fail2ban/jail.local`.
- Перед изменением известных конфигов создает резервные копии с timestamp.
- При выборе входа по ключу создает `~/.ssh/authorized_keys` для выбранного админ-пользователя.

## Быстрый старт

Подключитесь к новому VPS и запустите скрипт одной вставкой в терминал:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/4esyn/vps-security-bootstrap/main/setup-vps-security.sh)
```

Этот вариант подходит для чистого сервера, где файл `setup-vps-security.sh` еще не лежит в текущей директории.

Можно сначала посмотреть сценарий без изменений:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/4esyn/vps-security-bootstrap/main/setup-vps-security.sh) --dry-run
```

Быстрый старт предполагает, что на сервере доступны `bash` и `curl`. Для типового образа Ubuntu 24.04 VPS это нормальное ожидание.

Если скрипт обновил систему, предложил перезагрузку и вы перезагрузили сервер, после повторного подключения просто запустите ту же команду еще раз. Скрипт обнаружит сохраненное состояние и предложит продолжить с шага после обновления системы.

## Запуск из локальной копии

Если вы уже скопировали репозиторий или сам файл скрипта на сервер, используйте обычный локальный запуск:

```bash
chmod +x setup-vps-security.sh
./setup-vps-security.sh
```

Dry-run для локальной копии:

```bash
./setup-vps-security.sh --dry-run
```

## Какие вопросы задает скрипт

- Какой язык интерфейса использовать.
- Нужно ли сейчас обновить систему.
- Нужно ли сменить пароль `root`.
- Нужно ли создать или переиспользовать sudo-пользователя.
- Нужно ли настраивать SSH, менять порт и какой режим аутентификации использовать.
- Нужно ли сейчас настроить UFW.
- Нужно ли установить и настроить Fail2Ban.
- Нужно ли перезагрузить сервер, если система рекомендует reboot.

## Важные меры предосторожности

- Перед закрытием текущей SSH-сессии обязательно проверьте новый вход в отдельном терминале.
- Если выбираете режим только по ключу, убедитесь, что вставили правильный публичный ключ.
- Если меняете SSH-порт, скрипт сначала открывает новый порт и только потом предлагает удалить правило для `22/tcp`.
- Перед перезапуском SSH скрипт проверяет конфиг командой `sshd -t`.
- Текущая версия официально поддерживает только Ubuntu 24.04 LTS.
